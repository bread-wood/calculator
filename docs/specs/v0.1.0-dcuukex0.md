# Calculator CLI v0.1.x — Cold Start

**Theme:** Four-operator arithmetic from the command line. Pipeline proof-of-concept.

## Overview

A command-line calculator that evaluates arithmetic expressions passed as arguments and
prints the result to stdout. Targets developers and power users who want quick calculations
without leaving the terminal. Supports the four basic arithmetic operations with standard
operator precedence and parenthesized grouping. Serves as the Tier 1 proof-of-concept
for validating the breadmin-composer pipeline end-to-end.

---

## Success Criteria

- [ ] `calc '2 + 3'` prints `5` to stdout and exits 0
- [ ] `calc '10 / 4'` prints `2.5` to stdout and exits 0
- [ ] `calc '2 + 3 * 4'` prints `14` to stdout and exits 0
- [ ] `calc '(2 + 3) * 4'` prints `20` to stdout and exits 0
- [ ] `calc '4 / 2'` prints `2`, not `2.0` — no trailing `.0` when the result is whole
- [ ] `calc '1 / 0'` prints `error: division by zero` to stderr and exits 1
- [ ] `calc` with no arguments prints usage to stderr and exits 1
- [ ] `calc '2 +'` prints `error: unexpected end of expression` to stderr and exits 1
- [ ] `calc 'abc'` prints `error: unexpected token` to stderr and exits 1
- [ ] `make test` passes clean on macOS and Linux

---

## Scope

**Included:**

- Arithmetic: `+`, `-`, `*`, `/`
- Operator precedence (`*` and `/` before `+` and `-`)
- Parentheses for grouping
- Integer and floating-point number literals
- Clean integer output when the result has no fractional part
- Single-argument invocation: `calc '<expression>'`

---

## Failure Modes

**Input validation:**
- No arguments → print usage to stderr, exit 1
- More than one argument → print `error: expected a single quoted expression` to stderr, exit 1
- Empty string argument → print `error: empty expression` to stderr, exit 1
- Malformed expression → print `error: <description>` to stderr, exit 1
- Unrecognised token → print `error: unexpected token` to stderr, exit 1

**Arithmetic:**
- Division by zero → print `error: division by zero` to stderr, exit 1
- Result exceeds representable range → print `error: overflow` to stderr, exit 1

**No external service or subprocess boundaries for this product.**

---

## Interface Contracts

**CLI:**

```
calc '<expression>'
```

| Stream | Content |
|--------|---------|
| stdout | The result as a number (integer if whole, decimal otherwise), followed by a newline |
| stderr | `error: <description>` followed by a newline, on any failure |
| exit 0 | Expression evaluated successfully |
| exit 1 | Any error: malformed input, division by zero, wrong argument count, overflow |

---

## Constraints

- Runs on macOS and Linux with no external runtime dependencies beyond the standard library
- Output is a single line: the result or an error message
- No configuration files — all input via CLI argument
- Completes any valid expression in under 100 ms on commodity hardware
- The parser architecture must extend to named functions (`sin`, `sqrt`) and variables
  (`x = 5`) in future versions without a full rewrite

---

## Key Unknowns

   expressions planned for future versions — without requiring a parser rewrite when
   those features are added? The answer determines the core parser architecture.

## Non-Goals

- Variables and assignments (`x = 5`) — future version
- Named functions (`sin`, `sqrt`, `pow`) — future version
- Interactive REPL mode — future version
- Expression history or session state
- GUI or web interface
- Windows support in this version
