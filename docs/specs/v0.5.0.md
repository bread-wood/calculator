# Calculator CLI v0.5.x — Renderer

**Theme:** Expressions become pictures. The calculator draws.

## Overview

Adds a `plot` subcommand that evaluates an expression over a domain and writes
the result as an image file. Targets developers who want a quick function plot
without opening a spreadsheet or Python notebook. A single free variable `x`
sweeps the configured domain; the y-range is determined automatically from the
sampled values. This is the first version that produces visual output and
establishes the rendering pipeline that interactive versions will extend.

---

## Success Criteria

- [ ] `calc plot 'sin(x)'` writes a PNG to `./plot.png` and exits 0
- [ ] `calc plot 'sin(x)' --output out.svg` writes an SVG to `out.svg` and exits 0
- [ ] `calc plot 'x^2' --xmin -5 --xmax 5` plots `x^2` over `[-5, 5]` and exits 0
- [ ] `calc plot 'sin(x)'` produces an image where the curve crosses y=0 at x=0
  (verified by pixel sampling the output at the origin)
- [ ] `calc plot 'sin(x)' --width 800 --height 600` produces an 800×600 image and exits 0
- [ ] `calc plot '1 / x' --xmin -2 --xmax 2` skips the discontinuity at x=0 without
  crashing; the curve has a visible gap at x=0
- [ ] `calc plot 'unknown(x)'` prints `error: undefined function: unknown` to stderr
  and exits 1
- [ ] `calc plot` with no expression prints usage to stderr and exits 1
- [ ] `calc plot 'sin(x)' --output /nonexistent/dir/plot.png` prints
  `error: cannot write output: <reason>` to stderr and exits 1
- [ ] `make test` passes clean on macOS and Linux with no display required (image
  output only, no window)

## Scope

**Included:**

- `calc plot '<expression>'` subcommand
- Single free variable `x`; all other names resolved as constants or functions
- Configurable domain: `--xmin` and `--xmax` (default: `-10` to `10`)
- Automatic y-range: derived from sampled values; padded by 10% above and below
- Output formats: PNG and SVG selected by file extension on `--output`
- Default output: `./plot.png`
- Configurable image size: `--width` and `--height` (default: `800 × 600`)
- Axis lines and tick marks at round intervals
- Discontinuity handling: gaps in the curve where the expression is undefined or
  produces a domain error (no crash, no straight line across the gap)
- All v0.4.x expression features available in the plotted expression

---

## Failure Modes

**Expression errors:**
- Expression contains undefined variable other than `x` → `error: undefined variable: <name>` to stderr, exit 1
- Expression contains undefined function → `error: undefined function: <name>` to stderr, exit 1
- Expression is constant (no `x`) → renders a horizontal line; not an error

**Domain / range:**
- All sampled points produce domain errors (e.g. `sqrt(x)` with `--xmin -5 --xmax -1`) →
  `error: expression undefined over entire domain` to stderr, exit 1
- y-range collapses to a single value (constant expression) → renders a horizontal
  line at that value; not an error

**Output:**
- Output path not writable → `error: cannot write output: <reason>` to stderr, exit 1
- Unknown file extension on `--output` → `error: unsupported format: <ext>; use .png or .svg` to stderr, exit 1

**Input validation:**
- No expression argument → print usage to stderr, exit 1
- `--xmin` ≥ `--xmax` → `error: xmin must be less than xmax` to stderr, exit 1

## Interface Contracts

**CLI:**

```
calc plot '<expression>' [--xmin N] [--xmax N] [--width N] [--height N] [--output PATH]
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--xmin` | `-10` | Left edge of domain |
| `--xmax` | `10` | Right edge of domain |
| `--width` | `800` | Output image width in pixels |
| `--height` | `600` | Output image height in pixels |
| `--output` | `./plot.png` | Output file path; format inferred from extension (`.png`, `.svg`) |

| Stream | Content |
|--------|---------|
| stdout | Empty on success |
| stderr | `error: <description>` on any failure |
| exit 0 | Image written successfully |
| exit 1 | Any error |

---

## Constraints

- No display or windowing system required at runtime — image written to file only
- No external runtime dependencies beyond the standard library and any stdlib
  image/graphics packages available in the implementation language
- The rendering pipeline must be structured so that the same scene representation
  (axes, curve samples, labels) can be rendered to either a file or a live window
  in v0.7.x without duplicating the curve-generation logic
- Sampling resolution must be at least 1 sample per pixel of output width
- macOS and Linux; no Windows in this version

---

## Key Unknowns

1. **[P1]** Can a single rendering pipeline produce both file output (PNG/SVG) and
   live window output using the same scene representation — or do file-based and
   window-based rendering require fundamentally different approaches that cannot
   share the curve-generation layer? The answer determines whether v0.5.x lays
   the rendering foundation for the interactive versions or whether v0.7.x must
   replace it.

---

## Non-Goals

- Multiple curves on one plot — v0.6.x
- Parametric curves — v0.6.x
- Interactive window display — v0.7.x
- Expression entry in a window — v0.8.x
- Implicit curves or inequalities — v0.9.x
- Windows support
