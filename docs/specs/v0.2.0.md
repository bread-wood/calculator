# Calculator CLI v0.2.x — Function Library

**Theme:** Named functions and constants. The parser earns its extensibility.

## Overview

Extends the v0.1.x arithmetic calculator with named mathematical functions and
constants. Targets the same developer audience who want quick calculations in the
terminal, now covering the functions they reach for most — square root, trig,
rounding, exponentiation, and the fundamental constants `pi` and `e`. This version
validates that the v0.1.x parser architecture extends cleanly, serving as the
integration test for the extensibility constraint imposed in v0.1.x.

## Scope

**Included:**

- Single-argument functions: `sqrt`, `abs`, `floor`, `ceil`, `round`, `sin`, `cos`,
  `tan`, `log`, `exp`
- Two-argument functions: `pow`, `atan2`
- Named constants: `pi`, `e`
- Function calls composable with arithmetic and with each other: `sqrt(pow(3,2) + pow(4,2))`
- All v0.1.x arithmetic, precedence, and parenthesized grouping unchanged

---

## Failure Modes

**Function errors:**
- Unknown function name → `error: unknown function: <name>` to stderr, exit 1
- Wrong argument count → `error: wrong number of arguments: <name> expects <N>` to stderr, exit 1
- Domain error (e.g. `sqrt(-1)`, `log(0)`) → `error: domain error` to stderr, exit 1

**Input validation (inherited from v0.1.x):**
- No arguments → print usage to stderr, exit 1
- More than one argument → `error: expected a single quoted expression` to stderr, exit 1
- Empty string argument → `error: empty expression` to stderr, exit 1
- Malformed expression → `error: <description>` to stderr, exit 1
- Unrecognised token → `error: unexpected token` to stderr, exit 1

**Arithmetic (inherited from v0.1.x):**
- Division by zero → `error: division by zero` to stderr, exit 1
- Result exceeds representable range → `error: overflow` to stderr, exit 1

## Interface Contracts

**CLI:**

```
calc '<expression>'
```

| Stream | Content |
|--------|---------|
| stdout | The result as a number (integer if whole, decimal otherwise), followed by a newline |
| stderr | `error: <description>` followed by a newline, on any failure |
| exit 0 | Expression evaluated successfully |
| exit 1 | Any error: malformed input, unknown function, wrong argument count, domain error, arithmetic error |

**Function signatures:**

| Function | Arguments | Domain restriction |
|----------|-----------|--------------------|
| `sqrt(x)` | 1 | x ≥ 0 |
| `abs(x)` | 1 | none |
| `floor(x)` | 1 | none |
| `ceil(x)` | 1 | none |
| `round(x)` | 1 | none |
| `sin(x)` | 1 | none (radians) |
| `cos(x)` | 1 | none (radians) |
| `tan(x)` | 1 | none (radians) |
| `log(x)` | 1 | x > 0 |
| `exp(x)` | 1 | none |
| `pow(x, y)` | 2 | none |
| `atan2(y, x)` | 2 | none |

**Constants:**

| Name | Value |
|------|-------|
| `pi` | 3.141592653589793 |
| `e` | 2.718281828459045 |

## Constraints

- All v0.1.x constraints remain in force: macOS + Linux, stdlib only, single-line
  output, under 100 ms, no config files
- Functions are registered in a table; adding a new function requires adding one
  entry to the table, not modifying the parser
- Trig functions take arguments in radians; no degree mode
- `log` is the natural logarithm (base e); no `log2` or `log10` in this version
- The constant and function namespaces must remain extensible to user-defined
  variables (`x = 5`) in a future version without a parser rewrite

---

## Key Unknowns

1. **[P1]** Does the v0.1.x parser extend to function-call syntax and named
   constants without structural changes, and does the same extension point
   accommodate user-defined variables planned for a future version — or does
   adding function calls expose a limitation that requires rearchitecting the
   parser before variables can be added? The answer determines whether the
   v0.1.x extensibility constraint was satisfied or must be revisited before
   variables are scoped.

---

## Non-Goals

- Inverse trig functions (`asin`, `acos`) — not in common terminal use cases
- `log2`, `log10` — natural log covers the most common case
- Degree mode for trig functions
- Variables and assignments (`x = 5`) — future version
- User-defined functions
- Interactive REPL mode — future version
- Windows support in this version
